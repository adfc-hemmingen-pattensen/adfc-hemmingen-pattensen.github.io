Übersicht

Web-Seiten Generierung für https://adfc-hemmingen-pattensen.github.io/

Der Dreh- und Angelpunkt sind unformatierte Textseiten die im der Markdown Sprache geschrieben sind. Präziser im Dialekt “GitLab Flavored Markdown” (GFM).

Ein Markdown Beipiel:

#Markdown Überschrift
Der *schräge* Fuchs springt über den **fetten** Hund.
> Das ist kein Zitat

Eine Liste
* Punkt A
* Punkt B
* ![Ein Bild](https://www.adfc.de/static/images/logo_adfc.gif)

--------------------------------------------------------------

sieht dann später so aus:

Markdown Überschrift

Der schräge Fuchs springt über den fetten Hund.

Das ist kein Zitat

Eine Liste

  • Punkt A
  • Punkt B

Diese Seiten mit der Dateiendung .md werden mit dem Software Programm PanDoc in HTML Seiten konvertiert. Dieses geschieht auf der Kommando Zeile mit dem Befehl

pandoc seite.md -o seite.html #-o steht für Output

Zusammen mit den gespeicherten Bildern wird das ganze Verzeichnis mit dem Software Programm Git zuerst auf dem PC in eine Versionsablage kopiert (commit) und anschließend auf den Github Server geschoben (push). Hier befindet sich also eine komplette Kopie aller Versionen.

Der Github Server stellt anschließend die HTML und Bilder Dateien auf seinen Webserver unter der URL https://adfc-hemmingen-pattensen.github.io/ als Webseiten öffentlich zur Verfügung.

Nicht ohne Netz und doppelten Boden.

Das gute daran, die Dateien von dem Github Server kann jeder “klonen”, er hat dann die gleichen Dateien auf seinen Rechner. Er kann auch Änderungen wieder “pushen”, aber nur der Besitzer des Passwortes 😁 kann die Änderungen für die Öffentlichkeit freigeben. Oder anders gesagt, auch ich kann die Dateien, im Notfall, wieder bekommen.

Extras

Mängelkarte

Für unsere Mängelkarte benutze ich den umap Service. Die Karten werden direkt mithilfe des Browser auf den umap Server erstellt und bleiben dort liegen. Mithilfe des HTML iFrame Befehls wird unsere umap WebSeite im Browser des Betrachter eingebunden.

Terminübersicht und Tourenbeschreibungen

Für die Terminübersicht und Tourenbeschreibungen benutzen wir den HackMD Service. Dieses ist ein Online Markdown Editor mit einem eingebauten Markdown Betrachter (Pre-view). Jeder der die Links besitzt kann die Seiten ändern und erweitern. Die Inhalte dieser Seiten kopiere ich bei Bedarf manuell in unsere Markdown Seiten.

An eine teilweise automatische Einbindung von Inhalten, wie z.B. Toureneinladungen, ähnlich wie bei umap wird noch gearbeitet. (Stand 2018-04-20-Fri)

Grafiken

Sowohl HackMD als auch PanDoc, zusammen mit einem mermaid-filter, unterstützen die Grafiksprache mermaid. Ähnlich wie bei der Markdown Sprache besteht auch hier die Beschreibung und Formatierung rein aus unformatierten Texten.

Aus:

```mermaid
graph TD;
    A-->B;
    A-->C;
    B-->D;
```

wird:

Werkzeuge

Markdown lässt sich mit jedem Text Editor schreiben. Aber ähnlich wie bei HackMD, ist der Atom Editor, mit den Zusatzpaketen markdown-preview-enhanced und atom-mermaid der Editor meiner Wahl.

Für das schnelle Prüfen den nackten Markdown Seiten gibt es das Firefox Add-on GitLab Markdown Viewer. Dann kann man sich *.md Dateien direkt im Browser ansehen.
